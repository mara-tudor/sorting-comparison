# Sorting Algorithm Comparison

Companion repository for the paper:
**"An Experimental Comparison of Sorting Algorithms"**
— Mara Tudor, West University Timișoara, Department of Computer Science, 2026.

## Algorithms implemented

| Algorithm      | Best case    | Average case  | Worst case   | Space      |
|----------------|-------------|---------------|--------------|------------|
| Insertion Sort | O(n)        | O(n²)         | O(n²)        | O(1)       |
| Selection Sort | O(n²)       | O(n²)         | O(n²)        | O(1)       |
| Bubble Sort    | O(n)        | O(n²)         | O(n²)        | O(1)       |
| Merge Sort     | O(n log n)  | O(n log n)    | O(n log n)   | O(n)       |
| Quicksort¹     | O(n log n)  | O(n log n)    | O(n²)        | O(log n)   |
| Heap Sort      | O(n log n)  | O(n log n)    | O(n log n)   | O(1)       |
| Timsort        | O(n)        | O(n log n)    | O(n log n)   | O(n)       |

¹ Median-of-three pivot selection.

## How to reproduce the experiments

### Requirements

- Python 3.8 or newer (no external libraries required for benchmarking)
- `matplotlib` only if you want to generate plots

```bash
pip install matplotlib   # optional, for plots
```

### Run the benchmark

```bash
# Clone the repo
git clone https://github.com/mara-tudor/sorting-comparison.git
cd sorting-comparison

# Run benchmark (saves results/results.csv)
python benchmark.py

```

The script will:
1. Run a correctness check on all algorithms.
2. Time each algorithm over **15 trials** per (size, distribution) combination.
3. Save all raw results to `results/results.csv`.

### Input distributions tested

| Distribution  | Description                                              |
|---------------|----------------------------------------------------------|
| random        | Uniform random integers in [1, 10n]                      |
| sorted        | Elements 1, 2, …, n in ascending order                   |
| reverse       | Elements n, n−1, …, 1                                    |
| nearly_sorted | Sorted array with √n random adjacent swaps               |
| constant      | All elements equal to 1                                   |

### Array sizes

- **O(n log n) algorithms**: 100 · 500 · 1,000 · 5,000 · 10,000 · 50,000 · 100,000
- **O(n²) algorithms**: 100 · 500 · 1,000 · 5,000 · 10,000 (capped to avoid very long runtimes)

## Results

Raw results are available in [`results/results.csv`](results/results.csv).
Each row contains: algorithm name, distribution, array size, trial number, and time in seconds.
